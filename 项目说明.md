## ✨ 本修改版的新增功能

除沿用原 pi-web 全部功能外，本版本主要增加了三块内容：

### 1. 语音朗读（TTS）

让 agent 的回复"开口说话"，方便开车、做家务或眼睛需要休息时被动接收结果。

- **后端**：基于 Python `edge-tts` 生成语音，提供两个接口
  - `app/api/tts/stream/route.ts`：流式分段返回 base64 mp3，首段几乎秒出，长回复无需等待整段生成
  - `app/api/tts/route.ts`：非流式整段生成
- **前端 Hook**：`hooks/useTTS.ts`
  - `buildSpeechScript()`：把 Markdown 报告加工成口语稿——去掉格式符号、给标题加层级词、为表格/列表加引导语，避免"井号标题""竖线竖线"这种生硬朗读
  - 全局单例 store，支持队列播放、停止、状态订阅
- **自动朗读**：`components/ChatWindow.tsx` 中，agent 完成任务后（`agent_end` 事件）自动朗读最后一条助手消息，开头加一句"任务已完成"
- **手动朗读**：`components/MessageView.tsx` 每条消息底部有喇叭按钮，点击可朗读 / 再点停止

> 说明：本功能仅做"朗读输出"，不含语音输入/麦克风识别。

### 2. 背景颜色（主题切换）

顶栏新增调色板按钮，提供 5 套预设配色，一键切换界面氛围。

- 5 种主题：**白色 / 黑色 / 天蓝 / 薄荷绿 / 粉色**
- 实现：`hooks/useTheme.ts` 通过 CSS class（`dark` / `theme-sky` / `theme-mint` / `theme-pink`）切换，选择持久化到 `localStorage("pi-theme")`
- 切换动画：采用 `document.startViewTransition` + 圆形 `clip-path` 扩散过渡，从点击位置向外铺开新主题，过渡顺滑不突兀
- UI 入口：`components/AppShell.tsx` 顶栏的调色板弹层

### 3. 约束文件按钮（全局记忆与行为约束）

顶栏新增"全局记忆与行为约束"按钮，可直接在网页里编辑 PI 智能体自动读取的行为约束文件，无需开终端。

- 编辑对象：`~/AGENTS.md`（PI 在每次会话启动时会自动读取的全局约束文件，对所有项目生效）
- 编辑面板：`components/MemoryConfig.tsx`，内置默认模板，引导写入"希望 PI 遵守的规则"
- 后端：`app/api/memory/route.ts` 负责读写 `AGENTS.md`（同时兼容 `CLAUDE.md` / `MEMORY.md`，并区分项目级与全局级）
- 典型用法：把"回答用中文、代码加注释、先解释思路再写代码"等长期偏好写进 `~/AGENTS.md`，之后每个项目都自动遵守，不用每次重复叮嘱

## 🧱 其余说明（沿用原项目）

### 快速开始

```bash
# 直接运行（无需安装）
npx @agegr/pi-web@latest

# 或全局安装
npm install -g @agegr/pi-web
pi-web
```

启动后打开 [http://localhost:30141](http://localhost:30141)。

可选参数：

```bash
pi-web --port 8080              # 自定义端口
pi-web --hostname 127.0.0.1     # 仅本机访问
pi-web -p 8080 -H 127.0.0.1     # 组合使用
PORT=8080 pi-web                # 也支持环境变量
```

### 原有功能（继承自原 pi-web）

- **把历史工作接回来**：按项目浏览以往的 pi 对话，不必翻终端、记路径
- **放心试不同方向**：可从某条历史消息重新开始，也可 Fork 出独立新路线
- **边聊边看项目文件**：左侧浏览文件，右侧预览源码、文档、图片、音频、PDF、DOCX；文件变化自动刷新
- **随时掌握会话状态**：顶部显示上下文占用、花费、压缩结果、系统提示
- **少离开当前界面**：模型、登录/API key、模型测试、技能开关都能在网页里处理

### 数据目录与文件

- 默认读取 `~/.pi/agent/sessions` 下的会话文件，可用环境变量 `PI_CODING_AGENT_DIR` 指定其他目录
- 会话文件路径：`~/.pi/agent/sessions/<编码后的工作目录>/<时间戳>_<uuid>.jsonl`
- 模型配置：Models 面板读写 pi agent 目录下的 `models.json`
- 文件访问范围：当前所选项目目录，以及会话中出现过的工作目录
- Fork 创建新 `.jsonl` 文件；"Edit from here" 是同一会话文件内的分支

### 本地开发

```bash
npm install
npm run dev      # http://localhost:30141

# 检查
node_modules/.bin/tsc --noEmit
npm run lint
```

> 开发时不要运行 `next build` / `npm run build`，它会写入 `.next/`，影响正在运行的 dev server；构建留给发布流程。

### 语音功能额外依赖

本修改版的 TTS 依赖 Python 与 `edge-tts`，请确保运行环境已安装：

```bash
pip install edge-tts
```
